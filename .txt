1. Project Setup & Component Structure
First, you'll need a Remotion project.

Initialize Remotion Project:

npm init remotion my-crypto-explainer-video cd my-crypto-explainer-video npm install
Organize Your Code: A well-structured project is key for complex videos.

src/Root.tsx: This is your main Remotion entry point. It defines your video compositions.
src/Video.tsx: This component acts as the main orchestrator, sequencing all your individual scenes.
src/scenes/: Create a dedicated folder for each major scene. For example, Scene1_XRPIntro.tsx, Scene2_PriceCrash.tsx, etc. This keeps your scene logic encapsulated.
src/components/: This folder will house reusable UI elements and animation primitives. Examples include TextReveal.tsx, XRP_Coin.tsx, AbstractFigure.tsx, LineGraph.tsx, ImpactText.tsx.
src/assets/: Store all static assets like fonts, images (logos), SVG icons, and audio files here.
src/constants.ts: A file to define FPS, DURATION_IN_FRAMES, WIDTH, HEIGHT, and common colors.
Root.tsx Composition Example:

// src/Root.tsx import { Composition } from 'remotion'; import { MyCryptoExplainerVideo } from './Video'; import { FPS, DURATION_IN_FRAMES, VIDEO_WIDTH, VIDEO_HEIGHT } from './constants'; export const RemotionRoot: React.FC = () => { return ( <> <Composition id="CryptoExplainer" component={MyCryptoExplainerVideo} durationInFrames={DURATION_IN_FRAMES} fps={FPS} width={VIDEO_WIDTH} height={VIDEO_HEIGHT} // Add defaultProps here if you want to pass initial data /> {/* You can add more <Composition> components for individual scene previews */} </> ); };
2. Scene-Based Development & Precise Timing
This is where you bring your video's narrative to life, frame by frame.

Video.tsx - The Scene Orchestrator: This component uses Remotion's Sequence component to define when each scene starts and how long it lasts. // src/Video.tsx import { Sequence } from 'remotion'; import { FPS } from './constants'; import { Scene1_XRPIntro } from './scenes/Scene1_XRPIntro'; import { Scene2_PriceCrash } from './scenes/Scene2_PriceCrash'; // ... import all other scenes export const MyCryptoExplainerVideo: React.FC = () => { return ( <> {/* A global background or overlay for the entire video */} {/* For example, a shared 'HANO CRYPTO' logo in the corner */} {/* <GlobalLayout> */} <Sequence from={0} durationInFrames={FPS * 1}> {/* Scene 1: 1 second */} <Scene1_XRPIntro /> </Sequence> <Sequence from={FPS * 1} durationInFrames={FPS * 2}> {/* Scene 2: 2 seconds */} <Scene2_PriceCrash /> </Sequence> {/* Continue adding Sequences for each scene, ensuring 'from' matches the previous 'from + durationInFrames' */} <Sequence from={FPS * 3} durationInFrames={FPS * 5}> {/* Scene 3: 5 seconds */} {/* <Scene3_SEC_Gavel /> */} </Sequence> {/* ... and so on for all 12 scenes */} {/* </GlobalLayout> */} </> ); };
Frame-Based Timing Strategy: This is crucial for matching the fast-paced, synchronized style.
Voiceover First: Record or acquire your voiceover audio. This will be the backbone of your timing.
Detailed Transcription & Timing: Use an audio editing software (like Audacity, Adobe Audition, or even online tools) to get precise timestamps (down to milliseconds) for the start and end of every word or phrase in your voiceover.
Convert to Frames: Multiply these precise timestamps (in seconds) by your video's FPS (frames per second). This gives you the exact frame numbers for when each piece of audio starts and ends. This becomes your master timing sheet.
Orchestrate with Sequence and useCurrentFrame: Use these frame numbers to set the from and durationInFrames props for your Sequence components. Inside each scene component, use useCurrentFrame() to get the current frame number relative to the start of the composition, and interpolate() to animate elements based on these frame numbers.
3. Typography & Text Animations
The example video uses distinct text animations for impact and clarity.

Custom Fonts: Import your chosen modern, bold sans-serif fonts (e.g., Montserrat, Lato, Open Sans) into Remotion. You can do this by placing font files in src/assets/fonts/ and using CSS @font-face rules in your global CSS or directly within styled components.
TextReveal.tsx Component (Example - Word-by-Word/Fade): // src/components/TextReveal.tsx import React from 'react'; import { AbsoluteFill, useCurrentFrame, interpolate, spring, Easing } from 'remotion'; interface TextRevealProps { text: string; startFrame: number; // Frame relative to scene start durationFrames: number; delayPerWord?: number; // frames style?: React.CSSProperties; bounceEffect?: boolean; } export const TextReveal: React.FC<TextRevealProps> = ({ text, startFrame, durationFrames, delayPerWord = 0, style, bounceEffect = false, }) => { const frame = useCurrentFrame(); const words = text.split(' '); return ( <div style={{ ...style, position: 'absolute', width: '100%', height: '100%', display: 'flex', justifyContent: 'center', alignItems: 'center' }}> {words.map((word, index) => { const wordStartFrame = startFrame + index * delayPerWord; const wordEndFrame = wordStartFrame + durationFrames; const opacity = interpolate( frame, [wordStartFrame, wordStartFrame + 10], // Fade in over 10 frames [0, 1], { extrapolateLeft: 'clamp', extrapolateRight: 'clamp' } ); let scale = 1; if (bounceEffect) { scale = spring({ frame: frame - wordStartFrame, fps: FPS, // Your video's FPS config: { damping: 10, stiffness: 100 }, }); } if (frame < wordStartFrame || frame > wordEndFrame) { return null; } return ( <span key={index} style={{ opacity, transform: `scale(${scale})`, display: 'inline-block', // Important for word spacing margin: '0 5px', // Adjust spacing between words color: 'white', // Default color, can be overridden by style prop fontSize: 70, fontWeight: 'bold', textShadow: '0 0 10px rgba(255,255,255,0.5)', // Subtle glow whiteSpace: 'nowrap', // Prevent words from breaking }} > {word} </span> ); })} </div> ); };
Impactful Text (ImpactText.tsx): For words like "DIGITAL COMMODITY" or "Approved", use spring() for a strong scale-up with an overshoot (bounce) effect. Combine with filter: drop-shadow() for a glow.
Position Slide: Animate translateX or translateY properties using interpolate() for text sliding in from the sides or bottom (e.g., "Since 2018" rising).
4. Graphics & Icons
The video heavily relies on crisp, often glowing, graphical elements.

SVG Logos: Convert all crypto logos (XRP, BTC, ETH) and company logos (SEC, BlackRock, etc.) into SVG format. SVGs are ideal because they are vector-based, scale perfectly, and are easy to style and animate with CSS/React props.
Simulated 3D Look: The "minimalist 3D" effect is achievable without complex 3D libraries:
Layering with Offset: For coins, duplicate the SVG. Give the bottom layer a slightly darker color and offset its transform: translate() slightly to create a shadow/depth effect.
Gradients: Apply linear-gradient or radial-gradient to SVG fill properties to simulate curvature and lighting on flat surfaces.
Subtle Rotations: Use transform: rotateY() or rotateX() with interpolate() to give flat SVGs a sense of rotation or perspective.
Animating Logos (Example XRP Coin): // src/components/XRP_Coin.tsx import { AbsoluteFill, Img, useCurrentFrame, interpolate } from 'remotion'; import xrpLogo from '../assets/xrp-logo.svg'; // Ensure this path is correct import { FPS } from '../constants'; interface XRP_CoinProps { startRotationDeg?: number; endRotationDeg?: number; durationFrames: number; glowColor?: string; size?: number; } export const XRP_Coin: React.FC<XRP_CoinProps> = ({ startRotationDeg = 0, endRotationDeg = 360, durationFrames, glowColor = 'rgba(138, 43, 226, 0.8)', // Purple glow by default size = 200, }) => { const frame = useCurrentFrame(); const rotation = interpolate( frame, [0, durationFrames], [startRotationDeg, endRotationDeg], { extrapolateRight: 'loop', // For continuous rotation } ); return ( <AbsoluteFill style={{ justifyContent: 'center', alignItems: 'center' }}> <Img src={xrpLogo} style={{ width: size, height: size, transform: `rotateY(${rotation}deg)`, filter: `drop-shadow(0 0 ${size / 10}px ${glowColor})`, // Dynamic glow }} /> </AbsoluteFill> ); };
5. Glow Effects
The consistent use of glowing elements is a signature of this style.

CSS filter: drop-shadow(): This is your primary tool. Apply it to text, SVG elements, and other graphical components. filter: drop-shadow(0 0 10px rgba(138, 43, 226, 0.7)); /* Soft purple glow */ filter: drop-shadow(0 0 20px #00FF00); /* Bright green glow */
Multiple Layers: For a richer glow, you can stack multiple elements with different drop-shadow values (e.g., a small, intense shadow and a larger, softer, more diffuse shadow).
Opacity Animation: Animate the opacity of the drop-shadow or the element itself using interpolate() to create pulsing or fading glow effects.
6. Backgrounds & Gradients
The backgrounds are often dark and feature subtle gradients or patterns.

AbsoluteFill for Backgrounds: Use Remotion's <AbsoluteFill> component as your background layer within each scene.
CSS Gradients:
background: linear-gradient(...) for linear transitions (e.g., the dark red gradient in graph scenes).
background: radial-gradient(...) for circular light sources or effects (e.g., the green light in the ETF scene).
Dynamic Gradients: Animate the background-position or color stops of your gradients using interpolate() for subtle movement or color shifts.
Abstract Background Elements: For the blurry, semi-transparent XRP coins in the intro, render multiple XRP_Coin components with filter: blur() and reduced opacity. Animate their translateX/Y and scale for subtle background movement.
7. Data Visualizations (Graphs)
The line graphs are a key visual element for conveying data.

SVG path Element: This is essential for drawing custom line graphs. You'll define the d (path data) attribute of an SVG <path> element.
stroke-dasharray and stroke-dashoffset: Animate these CSS properties on an SVG <path> to create the "drawing" effect where the line appears to be drawn over time. // src/components/LineGraph.tsx import { AbsoluteFill, useCurrentFrame, interpolate } from 'remotion'; import { FPS } from '../constants'; interface LineGraphProps { pathData: string; // e.g., 'M0,100 L50,50 L100,80 ...' pathLength: number; // The total length of your SVG path startFrame: number; durationFrames: number; color?: string; strokeWidth?: number; } export const LineGraph: React.FC<LineGraphProps> = ({ pathData, pathLength, startFrame, durationFrames, color = 'white', strokeWidth = 4, }) => { const frame = useCurrentFrame(); const drawProgress = interpolate( frame, [startFrame, startFrame + durationFrames], [0, pathLength], { extrapolateLeft: 'clamp', extrapolateRight: 'clamp', } ); return ( <AbsoluteFill style={{ justifyContent: 'center', alignItems: 'center' }}> <svg width="100%" height="100%" viewBox="0 0 1000 500"> {/* Adjust viewBox as needed */} <path d={pathData} fill="none" stroke={color} strokeWidth={strokeWidth} strokeLinecap="round" strokeDasharray={pathLength} strokeDashoffset={pathLength - drawProgress} /> </svg> </AbsoluteFill> ); };
XRP Icon on Graph: To make an icon follow the graph line, you'll need to calculate its position along the SVG path at each frame. This can be done by getting points along the path based on the drawProgress.
8. Sound Integration
Audio is critical for pacing and impact in these videos.

Audio Component: Use Remotion's <Audio> component for background music and individual sound effects. // In your Video.tsx or a dedicated Audio component like src/VideoAudio.tsx import { Audio } from 'remotion'; import bgMusic from '../assets/bg-music.mp3'; import whooshSfx from '../assets/whoosh.mp3'; import hitSfx from '../assets/hit.mp3'; import { FPS } from './constants'; export const VideoAudio: React.FC = () => { return ( <> {/* Background music, looped and with adjusted volume */} <Audio src={bgMusic} loop volume={0.4} /> {/* Sound effects, precisely timed */} <Audio src={whooshSfx} startFrom={FPS * 0.9} endAt={FPS * 1.1} volume={1} /> {/* For Scene 1 to 2 transition */} <Audio src={hitSfx} startFrom={FPS * 3.5} endAt={FPS * 3.8} volume={0.8} /> {/* For gavel strike */} {/* Add more <Audio> components for each specific sound effect */} </> ); };
Sync with Visuals: Place <Audio> components at the exact startFrom and endAt frames determined by your timing sheet for sound effects. Often, sound effects should start slightly before or precisely with the visual action they accompany to enhance the impact.
9. Abstract Figures & Dynamic Shapes
AbstractFigure.tsx: This can be a simple combination of SVG shapes (e.g., a circle for the head, a rounded rectangle for the body) styled with gradients and drop-shadow for the glowing effect. Animate their opacity and scale for their appearance.
Geometric Shatter (Scene 2): This is visually impactful. You could:
Use SVG polygons that animate their points attribute to change shape, combined with opacity and scale for their appearance/disappearance.
Alternatively, generate many small div elements, give them white backgrounds, and animate their transform (position, rotation) and opacity to simulate rapid, sharp movements away from a central point.
10. Reusability and Modularity
Component-Based Design: Always identify common elements (e.g., TextReveal, XRP_Coin, AbstractFigure, ImpactText) and create dedicated, reusable components for them. This saves development time and ensures visual consistency across your video.
Props for Flexibility: Use React props to make your components highly configurable (e.g., TextReveal can take text, color, fontSize, startFrame, etc., as props).
11. Iteration and Debugging
Remotion Development Player: Use npm start frequently to open the Remotion Development Player. This allows you to preview your animations in real-time, scrub through frames, and quickly identify timing or animation issues.
Frame Control: The player's ability to jump to any frame is invaluable for fine-tuning easing curves and precise synchronization.
Performance: Be mindful of rendering performance. Complex animations or an excessive number of elements can slow down render times. Optimize by:
Using efficient CSS properties and transforms (e.g., transform is generally better than left/top).
Minimizing unnecessary re-renders of components.
Considering Remotion's freeze prop for static parts of your video.
